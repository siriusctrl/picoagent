Start one tool call or general-task subagent in the background and return immediately with a task_id. Direct tool calls begin in the foreground and automatically move to the same background-task lifecycle when their configured foreground window elapses. Use spawn when work is independent and the current task can continue safely. Do not background a mutation that later work depends on; call it directly or use task wait before consuming its result. Completed results are automatically appended at the next model boundary.
