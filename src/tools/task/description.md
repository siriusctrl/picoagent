Control background tools and subagents. status, wait, and stop return `{tasks:[{task_id,kind,name,status}]}`; wait is bounded and never stops work. inspect returns `{task_id,status,child_run_id,messages:[{seq,message}],has_earlier,next_before_seq}` (latest 6 messages by default); use next_before_seq as before_seq to page backward. steer queues an instruction after the child's current assistant/tool batch and returns `{task_id,child_run_id,status,input_id,message}`. Resolve every live task before the final answer; terminal results also arrive as durable background-result messages.
